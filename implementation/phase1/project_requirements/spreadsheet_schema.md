# Spreadsheet Data Store Specification

## Conventions
- One sheet tab per entity.
- First row is header; no merged cells.
- Every table has:
  - `Id`
  - `CreatedAtUtc`
  - `CreatedByUserId`
  - `UpdatedAtUtc`
  - `UpdatedByUserId`
  - `IsActive`
- All timestamps are UTC ISO-8601.
- IDs are generated by app service, not by formulas.

## Validation Rules
- `BidId` must exist before a `Project` can be created.
- `ProjectPhase` records auto-create from default template on project creation.
- A `WorkOrder` must belong to exactly one `ProjectPhase`.
- One invoice per work order (current business decision).
- Strict phase gating:
  - phase cannot move to `complete` unless:
    - all work orders in phase are `paid`
    - required inspections are `passed`
    - all required tasks are `done`

## Reporting Views (Derived)
- Bid pipeline by status and owner.
- Work orders by assignee and schedule window.
- Blocked work orders with root cause.
- Invoice aging by assignee/project.
- Actual vs estimate variance (from bid and invoice/work-order totals).
